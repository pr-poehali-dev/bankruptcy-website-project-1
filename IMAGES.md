# Управление изображениями

## Структура

Все изображения проекта управляются через файл `src/config/images.ts`.

## Компонент SafeImage

Компонент `SafeImage` автоматически обрабатывает ошибки загрузки изображений:

```tsx
import SafeImage from '@/components/SafeImage';
import { IMAGES } from '@/config/images';

<SafeImage
  src={IMAGES.hero.main}
  alt="Описание изображения"
  className="w-full h-full object-cover"
  fallbackIcon="Scale"  // Иконка для отображения при ошибке
  fallbackGradient="from-purple-100 to-orange-100"  // Градиент фона
/>
```

## Возможности

1. **Автоматический fallback**: Если изображение не загрузится, показывается красивая иконка с градиентом
2. **Lazy loading**: Изображения загружаются по мере прокрутки страницы
3. **Индикатор загрузки**: Показывается анимация пока изображение грузится
4. **Централизованное управление**: Все URL изображений в одном файле

## Добавление новых изображений

### Метод 1: Использование существующих изображений из IMAGES

1. Импортируйте конфиг:
```tsx
import { IMAGES } from '@/config/images';
```

2. Используйте через SafeImage:
```tsx
<SafeImage 
  src={IMAGES.blog.avoidBankruptcy} 
  alt="Как избежать банкротства"
  fallbackIcon="FileText"
  fallbackGradient="from-purple-100 to-blue-100"
/>
```

### Метод 2: Добавление новой категории изображений

1. Добавьте URL в `src/config/images.ts`:
```typescript
export const IMAGES = {
  // ... существующие категории
  myNewCategory: {
    main: 'https://cdn.poehali.dev/projects/.../image.jpg',
    fallback: '/images/fallback.svg'
  }
};
```

2. Используйте в компонентах:
```tsx
<SafeImage src={IMAGES.myNewCategory.main} alt="..." />
```

### Метод 3: Прямое использование URL

Для динамических изображений (например, из CMS):
```tsx
<SafeImage 
  src="https://cdn.poehali.dev/projects/.../dynamic-image.jpg"
  alt="Динамическое изображение"
  fallbackIcon="Image"
/>
```

## Что делать если изображение не создается?

Компонент SafeImage автоматически:
1. Показывает красивую иконку вместо сломанного изображения
2. Сохраняет дизайн страницы
3. Не ломает пользовательский опыт

Fallback работает на трех уровнях:
- **Уровень 1**: Основное изображение с CDN
- **Уровень 2**: Fallback иконка с градиентом (встроена в компонент)
- **Уровень 3**: Браузерный alt-текст

## Производительность

- Все изображения загружаются с CDN Poehali.dev
- Используется lazy loading для оптимизации
- Изображения кешируются браузером
- Fallback рендерится мгновенно без запросов
